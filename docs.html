Platform Guide

1. Introduction

Welcome to the platform documentation. This guide is for authorized partners and integrators who need direct access to the platform's transaction infrastructure through the desktop client and public API.

Key Features

Direct Cloud Execution

Server-side transaction processing through the cloud rather than physical terminal hardware.

Secure Token Authentication

Time-bound access tokens generated from the authenticated member dashboard.

Global Payout Network

Settlement to international bank accounts via SWIFT and to crypto wallets on supported networks.

Multi-Brand Support

The system can be switched between supported card brands within the client.

BIN Checker

Validate issuer, card type, and card level details before initiating a transaction.

Secure Proxy Network

All connections routed via a managed proxy network for transport-layer security.

Automatic Updates

The desktop application checks for and installs the latest version on launch.

Verified Application

Signed binaries ensure the application running on your machine is authentic.

2. System Architecture & Core Concepts

Understand the building blocks before integrating.

2.1 Authentication Flow (Access Token)

The platform does not use traditional username and password logins. It uses time-bound, limited-use access tokens generated from the member dashboard.

  • Preparation — operator prepares a source record via the web portal.
  • Authentication — operator pastes the access token into the client; the system validates against the configuration endpoint.
  • Consumption — once a new transaction starts, the token is consumed and cannot be reused. Prevents replay attacks.

2.2 Supported Transaction Protocols

The client supports the following internal protocol identifiers. Each describes a different transaction flow.

PROTO-A1 PROTO-A2 PROTO-A3 PROTO-A4 PROTO-A5 PROTO-A6 PROTO-B1 PROTO-B2 PROTO-B3

2.3 Global Payouts & Settlement

Configured in the client's Settings menu. Two settlement methods are supported:

  • Crypto Wallet Payout — split and send funds to receiver and sender addresses on a supported blockchain network.
  • Bank Payout — initiate a credit transfer to a designated international bank account.

2.4 BIN Checker

Retrieves non-sensitive metadata about a card's Bank Identification Number:

  • Issuing bank verification
  • Card type (Credit, Debit, Prepaid)
  • Card level (Classic, Gold, Platinum, Signature)

3. User Guide (Step-by-Step)

Step 1

Login and Authentication

Prepare a record in your member portal and copy the generated access token. Launch the desktop client, select your brand, paste your token, and click Login. The client validates the token and checks for updates.

Troubleshooting. If login fails: the token is mistyped, expired, already consumed, or the record was not prepared in the web portal.
Step 2

Main System Menu

After successful login you see four options:

  • NEW TRANSACTIONS — initiates a new flow. Consumes your token.
  • SYSTEM MAINTENANCE & UPDATES — checks for the latest version.
  • SETTINGS — configures default payout method.
  • BIN CHECKER — opens the BIN utility.
Step 3

Configure Payout Settings (Optional)

Recommended before your first transaction. Click Settings, choose Crypto Wallet Payout or Bank Payout, and save. Your selection determines which fields appear in the transaction flow.

Step 4

Perform a New Transaction

  1. Select the appropriate protocol from the dropdown.
  2. Enter the 16-digit card number prepared for this token.
  3. Enter the expiry date (MM/YY).
  4. Review fetched card details (cardholder name, issuing bank).
  5. Fill payout details (bank IBAN/SWIFT or wallet receiver/sender).
  6. Enter currency, amount, and settlement split.
  7. Enter the approval code (4 or 6 digits depending on protocol).
  8. Click Finalize Transaction.
Step 5

Processing and Report

The client processes the request and displays a final report with transaction status, transaction ID, and a digital signature for your records.

Step 6

Settlement

If approved, click Proceed to Settlement to see the fund distribution breakdown. After review, close the report and return to the login screen.

3. Integration Flow

The standard integration flow follows these steps:

  1. Prepare data record in the member dashboard.
  2. Call /get_token_details with the access code to confirm session data.
  3. Call /process_transaction with the required fields.
  4. Poll /transaction_status until a final state is returned.
  5. Retrieve settlement report via /settlement.

4. Get Token Details

Validates an access code and returns prepared session data.

POST /api/public/v1/get_token_details
Request
{
  "access_code": "YOUR_ACCESS_CODE",
  "member_id": 12345
}
Response
{
  "status": "success",
  "session": {
    "cardholder_name": "string",
    "issuing_bank": "string",
    "expires_at": "ISO-8601 timestamp"
  }
}

5. Process Transaction

Initiates a transaction with the provided parameters.

POST /api/public/v1/process_transaction
Request
{
  "access_code": "YOUR_ACCESS_CODE",
  "protocol": "PROTO-A1",
  "amount": 1000.00,
  "currency": "USD",
  "payout_method": "bank",
  "payout_details": { /* method-specific fields */ }
}
Response
{
  "status": "approved | declined | pending",
  "transaction_id": "TXN-...",
  "signature": "sha256-hash"
}

6. List Protocols

7. BIN Checker

Returns non-sensitive metadata for a given BIN.

POST /api/public/v1/bin_checker
Request
{
  "bin": "411111"
}
Response
{
  "bank": "string",
  "type": "credit | debit | prepaid",
  "level": "classic | gold | platinum | signature",
  "country": "ISO-3166 alpha-2"
}

8. Response Codes

400Bad RequestMissing or malformed parameter.

429Rate LimitedToo many requests. Back off and retry.
500Server ErrorInternal failure. Retry with exponential backoff.

9. Example Code

import requests

API_BASE = "https://api.your-brand.com/api/public/v1"

resp = requests.post(
    f"{API_BASE}/process_transaction",
    json={
        "access_code": "YOUR_ACCESS_CODE",
        "protocol": "PROTO-A1",
        "amount": 1000.00,
        "currency": "USD",
        "payout_method": "bank",
    },
)
print(resp.json())
const fetch = require("node-fetch");

const API_BASE = "https://api.your-brand.com/api/public/v1";

const res = await fetch(`${API_BASE}/process_transaction`, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    access_code: "YOUR_ACCESS_CODE",
    protocol: "PROTO-A1",
    amount: 1000.0,
    currency: "USD",
    payout_method: "bank",
  }),
});
console.log(await res.json());
