1. Introduction

Welcome to the official documentation. This guide covers system architecture, user workflows, and API integration for authorized partners.

Key Features

  • Direct Cloud Execution — server-driven transaction handling without dedicated terminal hardware.
  • Secure Authentication — time-bound access tokens issued from the member dashboard.
  • Global Payout Network — supports international bank transfers and digital wallet settlement.
  • Multi-Brand Support — switchable processing modes per network.
  • BIN Verification — issuer and card-type lookup before transaction submission.
  • Secure Proxy Network — routed traffic via vetted regional nodes.
  • Automatic Updates — version checks ensure clients run the verified build.

2. System Architecture & Core Concepts

2.1. Authentication Flow

The system uses time-bound access tokens instead of persistent passwords. Tokens follow a three-step lifecycle:

  1. Preparation — operator configures a session via the member dashboard.
  2. Authentication — token submitted to the client; validated against the configuration endpoint.
  3. Consumption — token is single-use; consumed upon first transaction to prevent replay.
